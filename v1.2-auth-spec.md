# v1.2 鉴权改造规格说明书

> 给 Claude Code 用。基于 ViralGenie 现有代码改造。

---

## 目标

把单一密码鉴权改成多用户系统,加 UI 中英文切换。

## 范围

### 做
1. AUTH_PASSWORD 改强密码(过渡保留)
2. Prisma 加 User 表(email, hashedPassword, name, locale, role)
3. NextAuth Credentials Provider 改成查 User 表(邮箱+密码登录)
4. 命令行脚本 `npm run add-user` 手动加用户
5. 现有 AnalysisTask.createdBy='admin' 数据迁移到管理员账号
6. UI i18n: 英文(默认)/ 中文,登录页 + 登录后均可切换
7. 用户偏好 locale 存 User 表,登录后默认按偏好

### 不做
- ❌ Google OAuth / Magic Link
- ❌ 用户自助注册
- ❌ 后台管理页面(用 Prisma Studio)
- ❌ 邮箱验证 / 二次认证
- ❌ AI 报告随语言切换(始终英文)
- ❌ 权限隔离(所有用户能看所有报告)

---

## 技术约束

- Next.js 16 / TypeScript / Prisma 7 / NextAuth.js
- 密码哈希: bcrypt
- i18n: next-intl 或 react-i18next 任选(选轻量的)
- 不引入新的大依赖

---

## Schema 改动

```prisma
model User {
  id             String   @id @default(cuid())
  email          String   @unique
  hashedPassword String
  name           String?
  locale         String   @default("en")  // "en" | "zh"
  role           String   @default("user") // "user" | "admin"
  createdAt      DateTime @default(now())
  updatedAt      DateTime @updatedAt
}
```

`AnalysisTask.createdBy` 字段保留 String 类型,迁移时把 'admin' 替换成管理员的 email 或 userId。

## 任务清单(按顺序)

### 1. 加密码哈希依赖
```bash
npm install bcryptjs
npm install -D @types/bcryptjs
```

### 2. 更新 Prisma schema
- 加 User 模型(见上)
- 本地: `npx prisma db push`
- 云端: 用现有的 DATABASE_URL 切换命令同步

### 3. 写加用户脚本
路径: `scripts/add-user.ts`
用法: `npm run add-user -- --email=xx@x.com --password=xxx --name=David --role=admin --locale=en`

要做的事:
- 检查 email 是否已存在
- bcrypt 哈希密码(salt rounds=10)
- 写入 User 表
- 输出确认信息

在 package.json 加:
```json
"scripts": {
  "add-user": "tsx scripts/add-user.ts"
}
```

### 4. 改 NextAuth 配置
文件: `src/lib/auth.ts`

Credentials Provider 的 authorize 函数改成:
- 接收 email + password
- 查 User 表
- bcrypt.compare 验证
- 验证通过返回 { id, email, name, role, locale }

JWT/Session callback 保留 role 和 locale 字段。

### 5. 改登录页
文件: `app/login/page.tsx`

- 表单加 email 字段(原本只有密码)
- 顶部加语言切换按钮(EN / 中)
- 登录页本身的文案根据当前语言显示
- 切换的语言存 cookie 或 localStorage

### 6. 加 i18n 系统
- 选 next-intl(Next.js 16 推荐)
- 创建 `messages/en.json` 和 `messages/zh.json`
- 翻译现有所有 UI 字符串(主页、报告页、Tab 标签、按钮等)
- 登录后用户的 locale 偏好覆盖 cookie

### 7. 加语言切换按钮(登录后)
- 主页右上角加一个 EN / 中 切换
- 切换时:
  - 更新 UI 立即生效
  - 调 API 更新 User.locale
  - 持久化

新 API: `PATCH /api/user/locale` { locale: "en" | "zh" }

### 8. 数据迁移
写一次性迁移脚本 `scripts/migrate-admin.ts`:
- 创建管理员账号(交互式问 email/密码,或读环境变量)
- 把所有 AnalysisTask.createdBy='admin' 更新成管理员的 email
- 跑一次后归档,不进 build

### 9. AUTH_PASSWORD 处理
- 现阶段保留环境变量 AUTH_PASSWORD,但**不用了**
- .env.local 和 Railway 里改成强密码(防止有缓存的旧 session 还在用)
- 一周后(确认所有 session 都走新登录)删掉这个变量
